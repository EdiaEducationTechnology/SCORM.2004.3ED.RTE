Known Issues

Known Issues

  1. If a package manifest contains references to SCOs that are located at an external website, a cross-domain JavaScript security error will occur. This may prevent the SCOs from communicating with the Sample RTE provided API.


  2. The Microsoft Internet Explorer Browser must be set to check for a new page every visit to the page. To configure the browser:
    • Open a Microsoft Internet Explorer Browser.
    • Click on "Tools" -> "Internet Options".
    • On the "General" tab, "temporary internet files" sub-section, click "Settings".
    • Ensure that the "Every visit to the page" radio button is selected.
    • Click "OK" and then "OK" again.

  3. It is possible that switching between different versions of the Sample RTE could cause an Applet Error if Applet Caching is enabled. If this occurs, the following steps can be taken to correct the issue in Java Versions 1.4 and later (up to but not including Version 5.0):
    • Enter Control Panel in your Windows environment.
    • Locate the Java Plug-in and double click on it.
    • Click on the "Cache" tab.
    • Click the "Clear" button in the upper right-hand corner.
    • Click Yes when prompted to confirm.
    • Uncheck the Enable Caching check box in the upper left-hand corner.
    • Click the Apply button.
    4. In Java Versions 5.0 and later, disable caching using the following steps:
    • Enter Control Panel in your Windows environment.
    • Open Java Control Panel.
    • Click on the "General" tab at the top of the dialog box.
    • Under the "Temporary Internet Files" section, click the "Settings" button.
    • Click the "View Applets" button.
    • Ensure that the check box labeled "Enable Caching" is unchecked (for the ADL applets).
