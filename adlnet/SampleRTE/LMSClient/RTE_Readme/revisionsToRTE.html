SCORM 2004 3rd Edition Sample RTE Version 1.0 Revisions
Changes from SCORM 2004 Sample RTE Version 1.3.3 to SCORM 2004 3rd Edition Sample RTE Version 1.0
  1. The Sample RTE now contains functionality to display API calls made by SCOs launched by the Sample RTE. To view the API calls that have been made by the SCOs, first click on the Activate Log link in the lower left corner of the Sample RTE and then click on the Show Log link. The log will be displayed in the frame immediately above the link. Hide Log will remove the display of the log string, but does not delete the log. Clear Log deletes the current contents of the log.
  2. The Sample RTE has been updated to hide the Suspend button when the Content Aggregation Content Package manifest explicitly states that the Suspend All control should be hidden.
  3. The Sample RTE has been updated to allow invisible activities to be the target of a Choice navigation event when no Table of Contents is rendered.
  4. The Sample RTE has been updated to display a validation error when importing a package in which there is a leaf item that contains no identifierref attribute or if the identifierref attribute is set to an empty string. In this case, the package may be imported with validation turned off. However, when running the course contained in that package, if the element with no identifierref attribute is reached, the ERROR page will be displayed.
  5. The Sample RTE has been updated to process an Exit All navigation request if the SCO sets cmi.exit to time-out. Any other pending navigation requests will be ignored.
  6. The Sample RTE Global Objectives Administration Page has been updated to include error checking for the measure value of a global objective. The acceptable range for measure is -1.0 to 1.0.
  7. The Sample RTE has been updated to recognize and ignore empty sequencing elements. If these elements occur in a SCORM Content Aggregation Content Package, the elements' defined defaults are used.
  8. The Sample RTE has been updated so that if a Suspend All navigation event is invoked and cmi.exit is not set to suspend, then SCORM Run-Time Environment Data Model infomation will set cmi.exit to suspend and persist the run-time data.
  9. The Sample RTE has been updated to properly implement the sequencing Selection Control element Selection Count. The Selection Control elements enable content developers to define sequencing information that indicates when to select certain activities and to limit the number of activities to be chosen. The Selection Count element limits the number of activities that are available.
  10. The Sample RTE has been updated to include an example implementation of IMS Shareable State Persistence SCORM Application Profile.
  11. The Sample RTE has been updated to properly handle several sets of non-English characters which may be used in the imsmanifest.
  12. The Sample RTE has been updated to allow Data Model setValue values to contain quotation marks and/or backslashes. Values containg any number of quotation marks and/or backslahses in any location in the value can be correctly displayed in the Data Model log.
  13. The Sample RTE SampleRTE.mdb database has been updated to allow the CourseTitle field of the CourseInfo table to store title strings of up to 200 characters in length as specified in the SCORM_CAM document.
  14. The Sample RTE has been updated to allow item identifiers to be distingusished by string letter-case as well as string text.
  15. The Sample RTE Data Model has been updated to allow time intervals containing sub-seconds to be correctly added and formatted between learner sessions.
  16. The Sample RTE has been modified to include links to Start Course and/or View Table of Contents for each course listed in the View Registered Courses menu. The links will allow the user to choose to begin the course and start sequencing by choosing Start Course if available, or view the Table of Contents and begin without starting sequencing by choosing View Table of Contents if available.
  17. The Sample RTE has been updated to utilize a proprietary file upload tool which has replaced the JSPSmartUpload tool.
  18. The course import process of the Sample RTE has been updated to display a validation error when importing a package in which the IMS Manifest contains an adlcp:persistState attribute. The adlcp:persistState attribute has been deleted.
  19. The course import process of the Sample RTE has been updated to no longer validate (sub)manifest elements with the inclusion of metadata within the (sub)manifest. A warning is now issued when a (sub)manifest is detected.
  20. The course import process of the Sample RTE has been updated to display a validation error when importing a package in which the IMS Manifest contains a referencedObjective attribute of a <ruleCondition> element that does not represent an identifier of an objective (<primaryObjective> / <objective>) found within that manifest.
  21. The course import process of the Sample RTE has been updated to enforce that when a <resource> is local to the content package, then a <file> element is required that represents the <resource>. The href attribute of the <file> element must be identical to the href attribute of the <resource>. The <file> element can exist as a direct child of the <resource> or can be referenced from the <dependency> element.
  22. The course import process of the Sample RTE has been updated to enforce that 0 or 1 <adlcp:completionThreshold> element exist as a child of an <item> element. The Sample RTE will display a validation error when more than one <adlcp:completionThreshold> element is detected as a child of the <item> element.
  23. Based on the IMS CP 1.1.4 updates, the course import process of the Sample RTE has been updated to enforce a Smallest Permitted Maximum (SPM) value of 2000 characters for an href that is pre-appended with xml:base value. This is a Best Practice warning only and does not affect validation.
  24. Based on the IMS CP 1.1.4 updates, the course import process of the Sample RTE has been updated to enforce excess baggage and dangling resource checks. Excess baggage is considered as files that exist in the Content Package that are not referenced from the IMS Manifest. Dangling resources elements are those that are not referenced by an "identifierref" attribute. These checks are Best Practice warnings only and do not affect validation.
  25. The course import process of the Sample RTE has been updated to display an error when the parameters attribute of an <item> element does not abide to the following syntax:
    • #<parameter>
    • <name>=<value>(&<name>=<value>)*(#<parameter>)
    • ?<name>=<value>(&<name>=<value>)*(#<parameter>)
  26. The course import process of the Sample RTE has been updated to prevent a java.lang.StringIndexOutOfBoundsException from occurring when a URI of 4 characters or less in length exists in an IMS Manifest.
  27. The course import process of the Sample RTE has been updated to conform to the requirement of SCORM 2004 3rd Edition that the element value equal "2004 3rd Edition". If the SCORM 2004 2nd Edition vocabulary token of "CAM 1.3" is detected, the course import process has been updated to display a warning with the ADL recommendation to update the value to "2004 3rd Edition". It is recommended that the content package manifest be updated because some systems may rely on this value to process the content package.
  28. The course import process of the Sample RTE has been updated to produce a validation warning when the xsi:schemalocation attribute in the root manifest declaration is not declared.
  29. When no xsi:schemalocation attribute is declared in the root manifest declaration, the Sample RTE provides the XML Parser with a defaulted list of controlling documents to use for XML validation. The course import process of the Sample RTE has been updated to test for only those controlling document(s) to exist at the root of the content package only if the respective namespace has been declared.
  30. The course import process of the Sample RTE has been updated to produce a warning when a URI exists multiple times in the IMS Manifest differing only in case.
  31. The course import process of the Sample RTE has been updated to support content package zip files that contain non-ASCII characters in the file name.
  32. The Sequencer was updated to ensure that objective state (local and global) is not affected by measure threshold evaluations during tree traversals.
  33. Numerous changes were made to the Sample RTE client to align implementation with the updated user interface interoperability requirements defined in the SCORM 2004 3rd Edition Conformance Requirements Document.
