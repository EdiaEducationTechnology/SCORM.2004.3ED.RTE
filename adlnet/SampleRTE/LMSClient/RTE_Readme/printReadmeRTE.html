SCORM 2004 3rd Edition Sample Run-Time Environment Version 1.0 Readme
Sample Run-Time Environment Introduction

Welcome to the SCORM 2004 3rd Edition Sample Run-Time Environment (RTE) Version 1.0 (Referred to hereafter as Sample RTE). The Sample RTE is intended to provide an example implementation of the concepts described in the SCORM. Specifically, it was developed in order to provide an example of the SCORM Run-Time Environment(API and Data Model) and SCORM Sequencing and Navigation as they might be applied in a Learning Management System (LMS) environment. Robust exception and error handling has not been included in this sample code.

Even though there are a number of valid Web-enabled SCORM implementations, this version of the Sample RTE was implemented as a Web-based client/server application using HTML, JavaScript, JavaServer Pages (JSP), Java Applets and Java Servlets (See the "Design Information" section of this Readme for additional Sample RTE design information).

Installation

It is recommended that you read through all of the Installation Instructions prior to beginning installation.

Prerequisites

This release of the Sample RTE requires the Sun Java 2 Software Development Kit, Standard Edition Version 1.4.2 or above. The Sun Java 2 Software Development Kit, Standard Edition is available for download free of charge at http://java.sun.com/products/archive/ At the time of release of this software, the latest version of the Sun Java 2 Software Development Kit, Standard Edition is 1.5.0_9.

Notes:
  • The Sample RTE requires the full J2SE SDK. The J2SE JRE alone is NOT sufficient for this software.
  • The Sample RTE client may be accessed from a machine other than the one on which it is installed using Microsoft Internet Explorer. In this case, the J2SE Runtime Environment (JRE) must be installed on the second machine.

After installing the J2SE SDK, you must do the following:

  1. Create an environment variable named "JAVA_HOME" and set its value to the root directory of the J2SE SDK.
    • For Windows 2000 and XP: Right click on "My Computer", select "Properties", select the "Advanced" tab and then click on "Environment Variables". Add a JAVA_HOME entry in the User or System Variables that points to the location where the J2SE SDK was installed (i.e., C:\j2sdk1.4.2_05).
  2. Add the full path of the J2SE SDK bin directory (where the java.exe is located) to the PATH environment variable.
    • For Windows 2000 and XP: Right click on "My Computer", select "Properties", select the "Advanced" tab and then click on "Environment Variables". Add the J2SE SDK bin directory to the Path by adding a semicolon followed by a reference to the JAVA_HOME environment variable followed by "\bin" (i.e., %JAVA_HOME%\bin).

Constraints

As of this release, the Sample Run-Time Environment has been tested on Microsoft Windows 2000 Professional (SP4) and Windows XP Professional SP2, both on the Intel Pentium PC platform with Microsoft Internet Explorer 6.0.

The Sample RTE has not been formally tested with any other browsers on any other platforms than those indicated above.

Installation

Installation of the Sample Run-Time Environment (RTE) is to be done only after installing the J2SE SDK as described in the Prerequisites.

The Sample RTE is available for download as a self-extracting executable install file. Once you have completed downloading the file, perform the following steps:

  1. Double-click on the downloaded self-extracting executable file to begin the installation process.
  2. Click the "Next" button to continue from the "Welcome" page.
  3. Read the "License Agreement" and
    • Click the "Yes" button to agree and continue with the install.
    • Click the "No" button if you do not agree with the terms of the license and wish to exit the install.
  4. The recommended root installation directory is "C:\ADL". You may choose another root installation directory, but the Sample RTE install will always append the "\ADL\SCORM_2004_3rd_Ed_Sample_RTE_V1.0" sub-directory to the specified root installation directory. The remainder of this Readme document assumes that the Sample RTE will be installed to "C:\ADL\C:\ADL\SCORM_2004_3rd_Ed_Sample_RTE_V1.0". This is referred to as "<drive>:\<install_directory>". If you override the default to use a different root installation directory, please take that into consideration when reading the remainder of this Readme.
  5. Click the "Next" button
  6. The install will add program icons to the Program Folder of your Start menu. You may type a new folder name, or select one from the existing list of Start Menu folders. It is recommended that you use "ADL". Then click on the "Next" button to install the Sample RTE. The installation program will copy the necessary files to the appropriate directories at this time.
  7. You will have the option to view a copy of this Readme document.
    • Check the "Show the Readme file" checkbox to view the Readme.
    • Uncheck the "Show the Readme file" checkbox to skip viewing the Readme.
  8. Click on the "Finish" button to complete the installation.
NOTE: The installation will not remove/un-install the Sample Run-Time Environment Version 1.3.3.

NOTE: The jakarta-tomcat-5.0.16 Servlet Engine and Web server will be installed and pre-configured to bind to port 8080.

Configuration

After installation, the following actions are necessary to ensure that the Sample RTE will run properly:
  • The Microsoft Internet Explorer browser must be set to check for a new page every visit to the page. To configure the browser:
    • Open a Microsoft Internet Explorer browser.
    • Click on "Tools" -> "Internet Options".
    • On the "General" tab, "Temporary Internet files" sub-section, click "Settings".
    • Ensure that the "Every visit to the page" radio button is selected.
    • Click "OK" and then "OK" again.

  • The Java Plug-in must be configured to disable caching. To disable the Java Plug-in cache:
    • For Java versions up to 1.5.0
      • Open the Windows "Control Panel".
      • Double click on the "Java Plug-in" icon.
      • Select the "Cache" tab.
      • Un-check the "Enable Caching" option.
      • Click "Apply".
      • Close Window


    • For Java versions after 1.5.0
      • Open the Windows "Control Panel".
      • Double click on the "Java Plug-in" icon.
      • Select the "General" tab.
      • Under the "Temporary Internet Files" section, click the "Settings" button.
      • In the "Temporary Files Settings" dialog box, click the "View Applets... " button.
      • In the "Java Applet Cache Viewer" dialog box, ensure the "Enable Caching" check box is unchecked for the Sample Run-Time Environment Applet(s).

  • Ensure that the Sun Java Plug-in is selected as the default Java Environment for the Microsoft Internet Explorer browser. To set the default browser Java Environment:
    • Open the "Java Plug-in" applet in the Control Panel
    • Select the "Browser" tab
    • Select the checkbox next to the browser that you are using to run the Sample RTE
    • Click "Apply"

  • If you are using Windows XP SP2 these additional actions are required:
    • Allow JavaScript to run by:
      • Open a Microsoft Internet Explorer browser
      • Click on "Tools" -> "Internet Options"
      • On the "Advanced" tab, check the box that states, "Allow active content to run in files on My Computer" under the "Security" section.
    • Allow the Sample RTE to pop-up windows by either disabling pop-up blocking or exclude the Sample RTE from pop-up blocking.

This completes the installation of the Sample RTE.

Known Issues

Known Issues

  1. If a package manifest contains references to SCOs that are located at an external website, a cross-domain JavaScript security error will occur. This may prevent the SCOs from communicating with the Sample RTE provided API.


  2. The Microsoft Internet Explorer Browser must be set to check for a new page every visit to the page. To configure the browser:
    • Open a Microsoft Internet Explorer Browser.
    • Click on "Tools" -> "Internet Options".
    • On the "General" tab, "temporary internet files" sub-section, click "Settings".
    • Ensure that the "Every visit to the page" radio button is selected.
    • Click "OK" and then "OK" again.

  3. It is possible that switching between different versions of the Sample RTE could cause an Applet Error if Applet Caching is enabled. If this occurs, the following steps can be taken to correct the issue in Java Versions 1.4 and later (up to but not including Version 5.0):
    • Enter Control Panel in your Windows environment.
    • Locate the Java Plug-in and double click on it.
    • Click on the "Cache" tab.
    • Click the "Clear" button in the upper right-hand corner.
    • Click Yes when prompted to confirm.
    • Uncheck the Enable Caching check box in the upper left-hand corner.
    • Click the Apply button.
    4. In Java Versions 5.0 and later, disable caching using the following steps:
    • Enter Control Panel in your Windows environment.
    • Open Java Control Panel.
    • Click on the "General" tab at the top of the dialog box.
    • Under the "Temporary Internet Files" section, click the "Settings" button.
    • Click the "View Applets" button.
    • Ensure that the check box labeled "Enable Caching" is unchecked (for the ADL applets).

SCORM 2004 3rd Edition Sample RTE Version 1.0 Revisions
Changes from SCORM 2004 Sample RTE Version 1.3.3 to SCORM 2004 3rd Edition Sample RTE Version 1.0
  1. The Sample RTE now contains functionality to display API calls made by SCOs launched by the Sample RTE. To view the API calls that have been made by the SCOs, first click on the Activate Log link in the lower left corner of the Sample RTE and then click on the Show Log link. The log will be displayed in the frame immediately above the link. Hide Log will remove the display of the log string, but does not delete the log. Clear Log deletes the current contents of the log.
  2. The Sample RTE has been updated to hide the Suspend button when the Content Aggregation Content Package manifest explicitly states that the Suspend All control should be hidden.
  3. The Sample RTE has been updated to allow invisible activities to be the target of a Choice navigation event when no Table of Contents is rendered.
  4. The Sample RTE has been updated to display a validation error when importing a package in which there is a leaf item that contains no identifierref attribute or if the identifierref attribute is set to an empty string. In this case, the package may be imported with validation turned off. However, when running the course contained in that package, if the element with no identifierref attribute is reached, the ERROR page will be displayed.
  5. The Sample RTE has been updated to process an Exit All navigation request if the SCO sets cmi.exit to time-out. Any other pending navigation requests will be ignored.
  6. The Sample RTE Global Objectives Administration Page has been updated to include error checking for the measure value of a global objective. The acceptable range for measure is -1.0 to 1.0.
  7. The Sample RTE has been updated to recognize and ignore empty sequencing elements. If these elements occur in a SCORM Content Aggregation Content Package, the elements' defined defaults are used.
  8. The Sample RTE has been updated so that if a Suspend All navigation event is invoked and cmi.exit is not set to suspend, then SCORM Run-Time Environment Data Model infomation will set cmi.exit to suspend and persist the run-time data.
  9. The Sample RTE has been updated to properly implement the sequencing Selection Control element Selection Count. The Selection Control elements enable content developers to define sequencing information that indicates when to select certain activities and to limit the number of activities to be chosen. The Selection Count element limits the number of activities that are available.
  10. The Sample RTE has been updated to include an example implementation of IMS Shareable State Persistence SCORM Application Profile.
  11. The Sample RTE has been updated to properly handle several sets of non-English characters which may be used in the imsmanifest.
  12. The Sample RTE has been updated to allow Data Model setValue values to contain quotation marks and/or backslashes. Values containg any number of quotation marks and/or backslahses in any location in the value can be correctly displayed in the Data Model log.
  13. The Sample RTE SampleRTE.mdb database has been updated to allow the CourseTitle field of the CourseInfo table to store title strings of up to 200 characters in length as specified in the SCORM_CAM document.
  14. The Sample RTE has been updated to allow item identifiers to be distingusished by string letter-case as well as string text.
  15. The Sample RTE Data Model has been updated to allow time intervals containing sub-seconds to be correctly added and formatted between learner sessions.
  16. The Sample RTE has been modified to include links to Start Course and/or View Table of Contents for each course listed in the View Registered Courses menu. The links will allow the user to choose to begin the course and start sequencing by choosing Start Course if available, or view the Table of Contents and begin without starting sequencing by choosing View Table of Contents if available.
  17. The Sample RTE has been updated to utilize a proprietary file upload tool which has replaced the JSPSmartUpload tool.
  18. The course import process of the Sample RTE has been updated to display a validation error when importing a package in which the IMS Manifest contains an adlcp:persistState attribute. The adlcp:persistState attribute has been deleted.
  19. The course import process of the Sample RTE has been updated to no longer validate (sub)manifest elements with the inclusion of metadata within the (sub)manifest. A warning is now issued when a (sub)manifest is detected.
  20. The course import process of the Sample RTE has been updated to display a validation error when importing a package in which the IMS Manifest contains a referencedObjective attribute of a <ruleCondition> element that does not represent an identifier of an objective (<primaryObjective> / <objective>) found within that manifest.
  21. The course import process of the Sample RTE has been updated to enforce that when a <resource> is local to the content package, then a <file> element is required that represents the <resource>. The href attribute of the <file> element must be identical to the href attribute of the <resource>. The <file> element can exist as a direct child of the <resource> or can be referenced from the <dependency> element.
  22. The course import process of the Sample RTE has been updated to enforce that 0 or 1 <adlcp:completionThreshold> element exist as a child of an <item> element. The Sample RTE will display a validation error when more than one <adlcp:completionThreshold> element is detected as a child of the <item> element.
  23. Based on the IMS CP 1.1.4 updates, the course import process of the Sample RTE has been updated to enforce a Smallest Permitted Maximum (SPM) value of 2000 characters for an href that is pre-appended with xml:base value. This is a Best Practice warning only and does not affect validation.
  24. Based on the IMS CP 1.1.4 updates, the course import process of the Sample RTE has been updated to enforce excess baggage and dangling resource checks. Excess baggage is considered as files that exist in the Content Package that are not referenced from the IMS Manifest. Dangling resources elements are those that are not referenced by an "identifierref" attribute. These checks are Best Practice warnings only and do not affect validation.
  25. The course import process of the Sample RTE has been updated to display an error when the parameters attribute of an <item> element does not abide to the following syntax:
    • #<parameter>
    • <name>=<value>(&<name>=<value>)*(#<parameter>)
    • ?<name>=<value>(&<name>=<value>)*(#<parameter>)
  26. The course import process of the Sample RTE has been updated to prevent a java.lang.StringIndexOutOfBoundsException from occurring when a URI of 4 characters or less in length exists in an IMS Manifest.
  27. The course import process of the Sample RTE has been updated to conform to the requirement of SCORM 2004 3rd Edition that the element value equal "2004 3rd Edition". If the SCORM 2004 2nd Edition vocabulary token of "CAM 1.3" is detected, the course import process has been updated to display a warning with the ADL recommendation to update the value to "2004 3rd Edition". It is recommended that the content package manifest be updated because some systems may rely on this value to process the content package.
  28. The course import process of the Sample RTE has been updated to produce a validation warning when the xsi:schemalocation attribute in the root manifest declaration is not declared.
  29. When no xsi:schemalocation attribute is declared in the root manifest declaration, the Sample RTE provides the XML Parser with a defaulted list of controlling documents to use for XML validation. The course import process of the Sample RTE has been updated to test for only those controlling document(s) to exist at the root of the content package only if the respective namespace has been declared.
  30. The course import process of the Sample RTE has been updated to produce a warning when a URI exists multiple times in the IMS Manifest differing only in case.
  31. The course import process of the Sample RTE has been updated to support content package zip files that contain non-ASCII characters in the file name.
  32. The Sequencer was updated to ensure that objective state (local and global) is not affected by measure threshold evaluations during tree traversals.
  33. Numerous changes were made to the Sample RTE client to align implementation with the updated user interface interoperability requirements defined in the SCORM 2004 3rd Edition Conformance Requirements Document.

Running the Sample Run-Time Environment (RTE) - Instructions

Running the Sample RTE

The instructions provided are for running the Sample RTE on a Windows platform. This is the only platform that the software has been tested on and is the only platform for which instructions are provided at this time.

To Run the Sample RTE:

First, start the jakarta-tomcat-5.0.16 servlet container and web server that is provided with this installation. To do this:

  • Click on the shortcut at Start Menu -> All Programs -> ADL -> SCORM_2004_3rd_Edition_Sample_Run-Time_Environment_version_1.0 -> Start Server. You will see a command prompt window appear and show several lines of textual information. This window will remain open until you stop the server.
  • Next click on the Start Menu -> All Programs -> ADL -> SCORM_2004_3rd_Edition_Sample_Run-Time_Environment_version_1.0 -> SCORM 2004 3rd Edition Sample Run-Time Environment version 1.0. This will launch the Sample RTE Main page. Log in with the username: "admin", password: "admin". (Be sure that Microsoft Internet Explorer 6.0 is your default browser.)

Online help pages are available once the Sample RTE is started. "Help" links appear at the bottom of each page.

To close the Sample RTE, first close the browser in which it is running. To stop the jakarta-tomcat-5.0.16 servlet container and web server, click on the Start Menu -> All Programs -> ADL -> SCORM_2004_3rd_Edition_Sample_Run-Time_Environment_version_1.0 -> Stop Server.

To return the Sample RTE to a "clean" state, log into the Sample RTE and from the main menu choose the Clear Database option. This will remove all course files that have been imported as well as any information associated with them, including IMS SSP buckets and related information; and will remove all user registration information. All Global Objective information will also be removed.

Design Information - Server Component

The Sample Run-Time Environment (RTE) Version 1.0 is composed of several distinct and separate components. These components are as follows:

  1. Sample Run-Time Environment Server Component: Implemented using Java Applications and Java Servlets.
  2. Run-Time Environment Client Component: Implemented using JSP, Java Applets, HTML, and JavaScript.


Sample Run-Time Environment Server Component

The Run-Time Environment Server Component is implemented using Java Servlets and Java Applications. The Servlets respond to requests from the Run-time Environment Client Component and are responsible for Data Model persistence as well as sequencing and parsing logic. The modules that make up the Server Component are listed below:

  • ADL Validator: This module is responsible for validating the imsmanifest.xml file that is required in SCORM Content Aggregation Content Packages. The ADL Validator is also responsible for parsing the XML file and returning a DOM tree representation to the Sample RTE Server Component. See the JavaDoc Package, "org.adl.validators" for details.
  • Utilities: This module contains general purpose utility Java files. See the JavaDoc Package, "org.adl.util" for details.
  • ADL Sequencer: This module is responsible for handling sequencing based on the IMS Simple Sequencing Specification and SCORM 2004 3rd Edition Sequencing and Navigation Version 1.0. It contains classes that create an initialized activity tree, maintain the activity tree throughout the sequencing process and handle all sequencing logic. See the JavaDoc Package, "org.adl.sequencer" for details.
  • Sample RTE: The Sample RTE Package is the hub of all of the Server Component packages. It contains the logic to utilize the other packages when needed. It also contains Java classes that interact with Java Server Pages to implement server side functionality. See the JavaDoc Packages,"org.adl.samplerte.server" and "org.adl.samplerte.util" for details.
  • Data Models: This package contains the classes that maintain the SCORM Run-Time Environment Data Model and the Navigation Data Model. See the JavaDoc Package "org.adl.datamodels" for details.
  • Shareable State Persistence: This package contains the classes used to provide the IMS Shareable State Persistence implementation. See the JavaDoc Package "org.ims.ssp.samplerte.server" for details.

Design Information - Client Component Java files

Sample Run-Time Environment Client Component (Java Files)

The Run-Time Environment Client Component consists of a sample user interface implemented in HTML and JavaScript and the Run-Time Environment API Instance implemented as a Java Applet. The Applet is downloaded to the client when the user accesses the Run-Time Environment page through a Web browser. This Applet provides the communication to the Run-time Environment Server Component for Data Model element persistence. See the JavaDoc Package, "org.adl.samplerte.client", for details.

Design Information - Client Component Web files

Sample Run-Time Environment Client Component (Web Files) is comprised of the following modules:

  • Administrative: This module contains files that change passwords, delete courses, manage global objectives, create or delete users, delete courses and clear the Sample RTE database. The files that make up the Administrative module are located in the "admin" folder.
  • Help: This module contains information pages on how to use the Sample RTE. The files that make up the Help module are located in the "help" folder.
  • Import: This module is responsible for managing the course import process. These files utilize the Sample Run-Time Environment Server side Java Component to handle parsing, validation and database access. The files that make up the Import module are located in the "import" folder.
  • Includes: The Sample RTE utilizes a style sheet (*.css) that handles the interface look and feel. See sampleRTE_style.css for details. This file is located in the "includes" directory under the Sample RTE Web Root.
  • Run-Time: This module contains the files used to deliver a SCORM Conformant Content Aggregation Content Package. In addition, the SCORM Application Programming Interface is included. All images used in the Sample RTE menu are located in the "menu-images" directory located in this module. The files that make up the Run-Time module are located in the "runtime" directory.
  • Special State: This module contains information pages that describe special sequencing states. It is possible that content will not be delivered as result of a sequencing request. In that event these pages inform the user about what state the sequencer is in. NOTE: These pages are included to help debug sequencing in content. There is no requirement by the SCORM that they be presented. LMSs may handle sequencing states in their own manner. The files that make up the Special State module are located in the "specialstate" folder under the Sample RTE Web Root.

Building the Sample Run-Time Environment (RTE)

Building the Sample RTE

NOTE: It is not a requirement to build the Sample RTE before using it. It will install as a fully operational unit. It is only necessary to build the Sample RTE if the source code has been changed.

In order to compile the Sample RTE, the full installation instructions must be completed first. The Sample RTE build file depends on a full jakarta-tomcat-5.0.16 install, along with resources provided in the installation of the RTE. It is important to note that when the RTE is built, it overwrites the previously installed Sample RTE. To move back to the original RTE, the un-install must be executed and then the downloaded self-extracting executable must be re-run.

The Sample RTE is built using Apache Ant, a Java-based build tool. Before building the Sample RTE, Ant must be installed. For information about and downloads of Ant, see http://ant.apache.org/.

To build the Sample RTE:
  • Open a command prompt and navigate to the directory where the Sample RTE is installed (Default: C:\ADL\SCORM_2004_3rd_Ed_Sample_RTE_V1.0).
  • From this point, navigate to the Sample_RTE\source directory.
  • At the prompt type "ant".

The XML Ant build files will compile all Java class and jar files needed by the Sample RTE. In addition the files will be installed to jakarta-tomcat-5.0.16. (NOTE: For changes to take effect, the tomcat server will need to be restarted). This completes the compilation of the Sample RTE.

Source File Locations

  • All JSP, HTML, and JavaScript files are located at <Sample RTE Install Dir>\Sample_RTE\source\SampleRTE\LMSClient
  • The Sample RTE Java files are located in their respective package locations. For example the data model classes are located at "org\adl\datamodels". The root of the Java source directory is <Sample RTE Install Dir>\Sample_RTE\source\src. Here you will find the build script and the root "org" directory.

Implementation Decisions

Case Sensitive URLs

  • URLs that are located in the imsmanifest.xml file and that identify launch locations and other hrefs are case sensitive in the Sample Run-Time Environment. For example, sco.html is different from SCO.HTML and from Sco.html. If the case used to specify a file name does not match the case of the actual file, an error will result while running the course.


Maximum Allowable Course Size

  • The maximum allowable size for a course imported into the Sample Run-Time Environment is approximately 2.0 gigabytes. Courses exceeding 2.0 gigabytes may not import correctly.


Manifest Encodings

  • Any course imported into the Sample Run-Time Environment which contains foreign characters in the manifest, must have the imsmanifest.xml file encoded in either UTF-8 or UTF-16 (Big Endian or Little Endian).
